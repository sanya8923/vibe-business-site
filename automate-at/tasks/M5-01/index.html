M5-01 Бот принимает голосовое — отвечает текстом

Интерн тренажёр сложность 1/5 free

Делаем простую автоматизацию: пользователь отправляет боту голосовое сообщение → n8n превращает голос в текст → бот отправляет этот текст обратно в чат. На первый взгляд всё просто — но именно эта задача показывает, что между нодами всегда передаются данные, и контролировать надо именно их.

К выполнению
Создай воркфлоу в n8n и собирай

В правом верхнем углу n8n — кнопка New workflow. Назови его M5-01 Транскрибатор и собирай по шагам ниже. Когда готов — жми «Сдать на ревью».

Минимальная мысль урока

На первый взгляд автоматизация выглядит так — три ноды в линию:

TGTelegram Trigger → AIOpenAI Transcribe → TGTelegram Send Message

Это базовая идея. Но в реальности такой схемы недостаточно — между этими шагами нужно добавить ещё два важных узла. Разберёмся почему.

Почему простой схемы недостаточно

⚠
Проблема 1
Telegram не передаёт сам аудиофайл

Telegram Trigger принимает сообщение и возвращает только служебную информацию о нём. В этой информации есть идентификатор файла — file_id.

Telegram Trigger как бы говорит: «Мне прислали голосовое, вот номер этого файла». А OpenAI не умеет распознавать «номер файла» — ему нужен сам аудиофайл.

Значит, между Trigger и OpenAI нужен дополнительный узел — Telegram Get File. Он берёт file_id и скачивает настоящий аудиофайл.

⚠
Проблема 2
Telegram Trigger срабатывает на любое сообщение

Триггер ловит всё подряд: текст, фото, стикеры, документы, голосовые. А наша автоматизация умеет работать только с голосом.

Если пользователь отправит, например, текст — следующий шаг начнёт искать голосовой файл, не найдёт его, и весь воркфлоу уйдёт в ошибку.

Чтобы этого не происходило, нужен фильтр. Он проверяет: в сообщении есть голос — идём дальше; нет — бот вежливо отвечает «Отправь голосовое». Роль фильтра возьмёт на себя нода IF.

Итоговая схема

После уточнения логики цепочка выглядит так:

TG Telegram Trigger любое сообщение
IF Это голос?
Да Нет
TG Get File скачать аудиофайл AI OpenAI Transcribe аудио → текст TG Send Message транскрипт в чат
TG Send Message «Отправь голосовое»
✓ Готово пользователь получил ответ

Простыми словами: бот получил сообщение → проверил, голос ли это → если да, скачал файл, распознал и отправил текст; если нет, попросил прислать голосовое.

Видео в разработке
Пока читай текст — видеоверсия появится позже.

Когда ты открываешь любой узел в n8n, ты видишь три зоны. Если научишься их различать — поймёшь n8n за один вечер. Если нет — будешь долго гадать, почему «не работает».

Слева
Input — что пришло
Что узел получил с предыдущего шага. Всегда JSON.
Центр
Настройки
Здесь ты говоришь узлу, что делать: тип операции, какие поля брать, как себя вести.
Справа
Output — что вернёт
Результат работы узла. То, что пойдёт дальше по цепочке.

Важный принцип: каждый узел берёт данные слева → обрабатывает → отдаёт результат направо. Если ты не понимаешь, что слева или что справа — ты не контролируешь процесс.

Шаги: собираем workflow

1 Telegram Trigger trigger

Что делает: ловит любое сообщение от пользователя.

Что настраиваем
  • Event Message

Больше ничего не нужно. Перед использованием — создай бота в Telegram через @BotFather.

📖 Как создать бота в Telegram через BotFather статья · ~2 мин ▾
▶
Видео в разработке

  1. Открой Telegram, найди @BotFather и нажми Start.

  2. Отправь команду /newbot. BotFather попросит имя бота и его username (должен заканчиваться на bot).

  3. В ответ ты получишь токен — длинная строка вида 123:ABC.... Это пароль твоего бота, не показывай его никому.

  4. В n8n открой ноду Telegram Trigger → Credentials → Create new → вставь токен. Готово.

Подробнее: статья в полной wiki.

2 IF — это голос? decision

Что делает: проверяет, есть ли в сообщении голосовой файл.

Что настраиваем
  • Value 1 {{ $json.message.voice }}
  • Operation Is Not Empty

Если голос есть — поток идёт по ветке true. Если нет — по ветке false: добавь Telegram Send Message с подсказкой пользователю.

💡
На заметку
Способов фильтрации больше одного

В M5-02 мы будем расширять воркфлоу на несколько типов файлов (audio, video, photo). И тогда станет понятно, что IF — самый простой выбор для одного условия, но не лучший, когда условий три-четыре. Тогда мы перейдём на Switch.

Видео по теме: сравнение IF / Filter / Switch.

Input ←
JSON из Telegram Trigger (со полем voice или без него).
Output →
Две ветки: true и false. В каждую — свои данные.
3 Telegram Get File в ветке true

Что делает: берёт file_id из сообщения и скачивает реальный аудиофайл с серверов Telegram.

Что настраиваем
  • Resource File
  • Operation Get
  • File ID {{ $json.message.voice.file_id }}
  • Download true

Поле File ID можно перетащить мышкой из левой панели — это удобнее, чем писать руками.

Input ←
{
  "message": {
    "voice": {
      "file_id": "ABC123"
    }
  }
}
Output →
{
  "binary": {
    "data": {
      "fileName": "voice.oga",
      "mimeType": "audio/ogg"
    }
  }
}
Появился блок binary — это и есть файл. До этого шага его не было.
4 OpenAI — Audio Transcribe в ветке true

Что делает: отправляет аудиофайл в Whisper. Модель принимает бинарные данные и возвращает текст.

Что настраиваем
  • Operation Audio Transcribe
  • Input Data Field data

Поле Input Data Field Name по умолчанию data — это имя бинарного свойства, которое Telegram Get File создаёт при скачивании. Если у тебя другое — измени.

⚠
Лимит файла
Whisper принимает файлы до 25 MB

Голосовые сообщения в Telegram обычно гораздо меньше — здесь это не проблема. Но для длинных аудио (M5-02) и видео (M5-03) лимит станет ограничением, и придётся резать файлы.

Input ←
Бинарный файл binary.data из предыдущего шага.
Output →
{
  "text": "Привет, как дела"
}
Готовый транскрипт.
5 Telegram Send Message обе ветки

Что делает: отправляет ответ пользователю в тот же чат, откуда пришло сообщение.

Что настраиваем (ветка true — отправляем транскрипт)
  • chat_id {{ $("Telegram Trigger").item.json.message.chat.id }}
  • text {{ $json.text }}
  • parse_mode HTML

В Options найди и выключи «Append n8n Attribution» — твоему боту это лишнее.

💡
Stretch внутри задачи
Сделай транскрипт копируемым в Telegram

В Telegram текст можно сделать «копируемым по тапу» — пользователь нажимает и текст моментально оказывается в буфере. Есть два HTML-способа: одним из них пользуйся в parse_mode=HTML.

Найди в интернете «как в HTML сделать текст копируемым» — выбери способ по вкусу и применяй в поле text.

Input ←
Текст из OpenAI Transcribe (ветка true) или подсказка для пользователя (ветка false).
Output →
Подтверждение Telegram, что сообщение отправлено.
Готов? Проверь себя

На каждом шаге задавай себе три вопроса:

  • что у меня сейчас в левой панели?
  • что появилось в правой?
  • что я передаю дальше?

Если ты это контролируешь — ты реально понимаешь n8n. Если нет — будешь долго гадать, почему «не работает».

Хочешь больше? Опциональные шаги ▾
  • Добавь поддержку отдельных аудиофайлов (audio) — это другое поле в объекте Telegram update, не voice.
  • Подсветь это в IF: проверь не только voice, но и audio. Прежде чем добавлять — посмотри что Telegram присылает в каждом случае.
