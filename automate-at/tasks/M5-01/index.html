M5-01 Бот принимает голосовое — отвечает текстом

Интерн тренажёр сложность 1/5 free

Делаем простую автоматизацию: пользователь отправляет боту голосовое сообщение → n8n превращает голос в текст → бот отправляет этот текст обратно в чат. На первый взгляд всё просто — но именно эта задача показывает, что между нодами всегда передаются данные, и контролировать надо именно их.

Минимальная мысль урока

На первый взгляд автоматизация выглядит так — три ноды в линию:

TGTelegram Trigger → AIOpenAI Transcribe → TGTelegram Send Message

Это базовая идея. Но в реальности такой схемы недостаточно — между этими шагами нужно добавить ещё два важных узла. Разберёмся почему.

Почему простой схемы недостаточно

⚠
Проблема 1
Telegram не передаёт сам аудиофайл

Telegram Trigger принимает сообщение и возвращает только служебную информацию о нём. В этой информации есть идентификатор файла — file_id.

Telegram Trigger как бы говорит: «Мне прислали голосовое, вот номер этого файла». А OpenAI не умеет распознавать «номер файла» — ему нужен сам аудиофайл.

Значит, между Trigger и OpenAI нужен дополнительный узел — Telegram Get File. Он берёт file_id и скачивает настоящий аудиофайл.

⚠
Проблема 2
Telegram Trigger срабатывает на любое сообщение

Триггер ловит всё подряд: текст, фото, стикеры, документы, голосовые. А наша автоматизация умеет работать только с голосом.

Если пользователь отправит, например, текст — следующий шаг начнёт искать голосовой файл, не найдёт его, и весь воркфлоу уйдёт в ошибку.

Чтобы этого не происходило, нужен фильтр. Он проверяет: в сообщении есть голос — идём дальше; нет — бот вежливо отвечает «Отправь голосовое». Роль фильтра возьмёт на себя нода IF.

Итоговая схема

После уточнения логики цепочка выглядит так:

TG Telegram Trigger любое сообщение
IF Это голос?
Да Нет
TG Get File скачать аудиофайл AI OpenAI Transcribe аудио → текст TG Send Message транскрипт в чат
TG Send Message «Отправь голосовое»
✓ Готово пользователь получил ответ

Простыми словами: бот получил сообщение → проверил, голос ли это → если да, скачал файл, распознал и отправил текст; если нет, попросил прислать голосовое.

Когда ты открываешь любой узел в n8n, ты видишь три зоны. Если научишься их различать — поймёшь n8n за один вечер. Если нет — будешь долго гадать, почему «не работает».

Слева
Input — что пришло
Что узел получил с предыдущего шага. Всегда JSON.
Центр
Настройки
Здесь ты говоришь узлу, что делать: тип операции, какие поля брать, как себя вести.
Справа
Output — что вернёт
Результат работы узла. То, что пойдёт дальше по цепочке.

Важный принцип: каждый узел берёт данные слева → обрабатывает → отдаёт результат направо. Если ты не понимаешь, что слева или что справа — ты не контролируешь процесс.

Видео в разработке
Пока читай текст — видеоверсия появится позже.

Триггер — это нода, с которой начинается воркфлоу. Без триггера воркфлоу не запустится сам — придётся каждый раз жать «Test workflow» вручную. Триггер ловит событие во внешнем мире (новое сообщение в Telegram, заполнение формы, расписание) и стартует цепочку.

  1. В пустом воркфлоу нажми + Add first step… — откроется панель добавления.

  2. Выбери раздел «On a trigger event» или ищи нужный триггер по названию (Telegram, Webhook, Schedule, Form).

  3. Подключи credentials (если нужны). Для Telegram — токен от @BotFather; для Webhook — credentials не нужны.

  4. Нажми Listen for test event и выполни действие во внешнем мире (отправь сообщение боту, открой форму, дождись расписания). Триггер поймает событие и покажет JSON в правой панели — это и есть Output.

В этой задаче мы ставим Telegram Trigger с событием Message. Все остальные триггеры устроены по тому же принципу — отличается только источник события.

⚙ Как выглядят настройки Get File в интерфейсе статья · ~1 мин ▾

Открой ноду — в центральной панели увидишь четыре поля:

  • Resource — что ты делаешь (File / Chat / Message…). Для скачивания файла — File.
  • Operation — какое действие. Get — забрать файл по идентификатору.
  • File ID — идентификатор файла из входящего сообщения. Перетащи поле мышкой из левой панели.
  • Download — переключатель true: только при включённом скачивании появятся бинарные данные на выходе. Если оставить false, нода вернёт только метаданные файла, без содержимого.
📦 Что такое binary в n8n статья · ~2 мин ▾

В n8n каждый «item» (объект данных) состоит из двух частей:

  • json — обычные данные в формате JSON: текст, числа, объекты, массивы. Это всё, что можно прочитать глазами.
  • binary — бинарные данные: файлы, картинки, аудио, PDF. Это содержимое файла, его нельзя «распечатать» в JSON-виде. Хранится отдельным блоком.

Когда нода Telegram Get File скачивает аудио — она кладёт его в binary.data. В правой панели Output ты увидишь не текст, а превью: имя файла, mime-тип, размер.

Важно: ноды, которые работают с файлами (OpenAI Transcribe, конвертеры, отправка вложений), читают именно binary, а не json. Если передаёшь файл — убедись, что bin-блок есть.

🗣 Что такое Whisper статья · ~2 мин ▾

Whisper — это модель распознавания речи от OpenAI. Принимает аудио на входе, отдаёт текст на выходе. Поддерживает 90+ языков, включая русский.

В ноде OpenAI с операцией Audio Transcribe запрос идёт именно в Whisper — n8n не использует другие движки для этой операции.

Что про неё стоит знать:

  • Принимает форматы mp3, mp4, m4a, wav, webm, oga (то, что присылает Telegram).
  • Сама определяет язык — указывать вручную не обязательно.
  • Возвращает только текст. Без таймкодов, без разделения дикторов (для этого есть отдельные модели).

5 Telegram Send Message обе ветки

Что делает: отправляет ответ пользователю в тот же чат, откуда пришло сообщение.

Что настраиваем (ветка true — отправляем транскрипт)
  • chat_id {{ $("Telegram Trigger").item.json.message.chat.id }}
  • text {{ $json.text }}
  • parse_mode HTML

В Options найди и выключи «Append n8n Attribution» — твоему боту это лишнее.

⚙ Как выглядят настройки Send Message в интерфейсе статья · ~1 мин ▾

Открой ноду — в центральной панели три ключевых поля и блок Options снизу:

  • Resource Message · Operation Send Message — это две стандартные настройки, отвечающие за «отправить ответ».
  • Chat ID — куда отправлять. Бери из исходного триггера: {{ $("Telegram Trigger").item.json.message.chat.id }}. Если возьмёшь его из текущего шага — в ветке false его может не быть.
  • Text — что отправлять. Из ветки true подставляй {{ $json.text }} (транскрипт), из ветки false — статичный текст «Отправь голосовое».
  • Additional Fields → Parse Mode — переключи на HTML, чтобы работали HTML-теги (нужно для копируемого текста).
  • Options → Append n8n Attribution — выключи. По умолчанию n8n добавляет в конец сообщения свою подпись, в работающем боте это не нужно.

Найди в интернете «как в HTML сделать текст копируемым» — выбери способ по вкусу и применяй в поле text.
