M0-01 Знакомство с n8n: первый воркфлоу

Интерн модуль сложность 1 free

Что делаем

Соберём первый рабочий воркфлоу в n8n. По кнопке Test workflow воркфлоу запускается, идёт в HTTP-ноду, забирает там JSON и возвращает его. Простейшая цепочка из двух нод — но это база, без которой дальше никак.

Что должно получиться

После сдачи задачи у тебя в n8n должен быть воркфлоу из двух нод:

  • Manual Trigger — запускается по кнопке
  • HTTP Request — делает GET на тестовый эндпоинт https://httpbin.org/get

После запуска во второй ноде в выходных данных должен прийти JSON со структурой { "args": {}, "headers": {...}, "url": "..." }. Проверь что воркфлоу работает — это главное.

Шаги

  1. Зайди в n8n по логину справа.
  2. Создай новый воркфлоу. Назови его M0-01 Первый WF.
  3. Добавь Manual Trigger.
  4. После триггера добавь HTTP Request, метод GET, URL https://httpbin.org/get.
  5. Нажми «Test workflow». Должно зелёное «выполнено».
  6. Открой выходные данные HTTP Request — посмотри JSON.
  7. Сохрани воркфлоу. Сдай на ревью кнопкой справа.

Если застрял

Если кнопка «Test workflow» серая — значит воркфлоу пустой. Проверь, что обе ноды на canvas и соединены линией. Если HTTP Request падает с ошибкой — в Settings ноды поставь Timeout 10000 ms.
