S-Embed-Cohere Замени движок поиска на Cohere — агент заговорит по-русски

Джуниор каталог AI-01 стретч сложность 4/10

Заменяем embedding-модель в двух воркфлоу — с text-embedding-3-small (OpenAI) на embed-v4.0 (Cohere). Одно изменение, два воркфлоу, одна тонкость с параметром. Результат: поиск по русскоязычной базе знаний становится на 15–20% точнее.

Стало Cohere embed-v4.0 (1024d) — специально обученная на 100+ языках, русский на уровне с английским

Что меняем и зачем

Когда агент ищет ответ в базе знаний, он сначала превращает вопрос пользователя в вектор — цифровое представление смысла. Embedding-модель — это и есть тот инструмент, который делает превращение. По умолчанию мы используем модель OpenAI.

Проблема: OpenAI text-embedding-3-small обучалась преимущественно на английских текстах. Русский язык со всей его морфологией — «стол», «стола», «столу», «столы» — для неё сложнее. Cohere embed-v4.0 специально создавалась для 100+ языков и понимает русский так же хорошо, как английский.

⚠
Когда это проявляется
Агент находит не то или «не знает» ответа — хотя он есть в базе

Пользователь спрашивает «где находятся наши склады» — в базе есть документ про «склад» и «хранение». OpenAI embedding иногда не свяжет «склады» и «склад» как близкие по смыслу. Cohere embed-v4 с этим справляется — она понимает морфологию и синонимы на русском языке.

На бенчмарках: на русскоязычных запросах Cohere embed-v4 показывает на 15–20% лучший recall, чем OpenAI text-embedding-3-small.

Что меняется в воркфлоу

Два воркфлоу содержат sub-ноду Embeddings OpenAI. Меняем её на Embeddings Cohere — с важным отличием в параметре между ними.

БЫЛО
WF1 / WF2 — Embeddings OpenAI
Model: text-embedding-3-small
Размерность: 1536d
→
СТАЛО
WF1 = search_document / WF2 = search_query
Model: embed-v4.0
WF1 Input Type: search_document
WF2 Input Type: search_query ← !
Размерность: 1024d
💡
Главная тонкость
В WF1 и WF2 параметр Input Type разный

В WF1 ставим search_document — это режим для индексации документов. В WF2 ставим search_query — это режим для поиска по запросу пользователя. Если поставить везде одинаково — качество поиска упадёт на 5–10%. Разбор причины — в статье ниже.

Создай credential в n8n

🔑 Пошаговая инструкция: добавить Header Auth credential в n8n статья · ~3 мин ▾

  1. В левом меню n8n нажми Settings (шестерёнка) → Credentials.

  2. Нажми Add credential (кнопка в правом верхнем углу).

  3. В поиске набери Header Auth → выбери Header Auth.

  4. Заполни поля:

    • Credential name: Cohere API (именно так — ноды будут искать по этому имени)
    • Name: Authorization
    • Value: Bearer твой-api-key (подставь ключ из шага 0.2, пробел между Bearer и ключом обязателен)

  5. Нажми Save. Credential появится в списке и станет доступен во всех нодах.

Почему Header Auth, а не «Cohere API»?
Cohere Embedding API принимает ключ через HTTP-заголовок Authorization: Bearer ключ. Header Auth — это именно такой универсальный способ. В некоторых версиях n8n есть встроенный тип Cohere API — если видишь его в поиске, используй его: он удобнее. Если нет — Header Auth работает одинаково хорошо.

Шаги внедрения

1 Пересоздай Qdrant-коллекцию делать ПЕРВЫМ

Почему первым: Qdrant хранит коллекцию с фиксированной размерностью. OpenAI делал векторы на 1536 чисел, Cohere — на 1024. Эти два формата несовместимы. Если сначала поменяешь ноды, а потом попытаешься загрузить файл — Qdrant вернёт ошибку. Поэтому сначала удаляем старую коллекцию, создаём новую, а потом меняем ноды.

📐 Почему нельзя смешивать векторы разной размерности статья · ~3 мин ▾

Когда ты создаёшь коллекцию в Qdrant, ты один раз говоришь: «в этой коллекции векторы будут размером 1536». Это прошивается в структуру БД навсегда — как тип колонки в SQL-таблице.

Embedding-модели выдают векторы конкретного размера. Это не случайность — размер определяет «сколько смысла» помещается в вектор:

  • OpenAI text-embedding-3-small → 1536 чисел
  • Cohere embed-v4.0 → 1024 числа (или меньше через Matryoshka)
  • bge-m3 → 1024 числа

Если ты попытаешься записать 1024-мерный вектор в коллекцию, созданную под 1536 — Qdrant откажет с ошибкой Wrong input: expected dim 1536, got 1024.

Что делать:

  1. Удалить старую коллекцию (все векторы пропадут).
  2. Создать новую с size: 1024. Это делается автоматически при первом insert'е через ноду Qdrant Insert.
  3. Перезалить все файлы KB через WF1 — теперь они запишутся с правильными 1024-мерными векторами.

Боятся ли потери данных? Нет. KB — это производное от твоих исходных файлов. Файлы у тебя есть, WF1 умеет их грузить. Удаление коллекции = «очисти индекс», а не «удали документы».

Как удалить коллекцию через Qdrant HTTP Request

Самый простой способ — выполни HTTP Request прямо в n8n (один раз, вручную, без сохранения в WF):

  • Method DELETE
  • URL https://qdrant.school.vibe-business.site/collections/c01-kb-agent
  • Header api-key: <твой Qdrant Master ключ>

Замени c01 на свой student_id. Qdrant Master credentials лежат в n8n — ключ смотри там.

До удаления
Коллекция c01-kb-agent с векторами 1536d. Файлы KB уже залиты через OpenAI embedding.
После удаления
Коллекции нет. При следующем запуске WF1 — Qdrant Insert создаст новую с размерностью 1024 (под Cohere). Сохранять старые данные не нужно.
2 WF1 — замени Embeddings OpenAI → Cohere Chat Trigger Ingestion

Что делаем: в нодах Qdrant Insert TXT и Qdrant Insert PDF есть sub-нода Embeddings OpenAI. Заменяем её на Embeddings Cohere.

Embeddings OpenAI (text-embedding-3-small) → Embeddings Cohere (embed-v4.0)
Как заменить sub-ноду
  1. Открой WF1 Chat Trigger Ingestion.
  2. Кликни на ноду Qdrant Insert TXT → в правой панели найди sub-ноду Embeddings OpenAI.
  3. Нажми × на sub-ноде, чтобы отсоединить её.
  4. Нажми + (Add document embedding) → выбери Embeddings Cohere.

🔀 Что такое sub-нода и как она работает внутри Qdrant Insert статья · ~2 мин ▾

В n8n AI-ноды типа Qdrant Insert, Qdrant KB, AI Agent устроены как «родитель + дети». Родитель — основная нода (например, Qdrant Insert — записывает векторы в БД). Дети — sub-ноды, которые подключаются к кружкам на нижней стороне родителя.

Sub-нода Embeddings OpenAI — это «провайдер эмбеддингов». Когда Qdrant Insert хочет записать чанк текста, он обращается к этой sub-ноде: «переведи мне этот текст в вектор». Sub-нода делает запрос в OpenAI API и возвращает вектор.

Когда ты меняешь sub-ноду на Embeddings Cohere — ты просто говоришь Qdrant Insert: «теперь за векторами иди не в OpenAI, а в Cohere». Всё остальное в WF1 остаётся нетронутым.

🎯 Зачем Input Type разный в WF1 и WF2 статья · ~3 мин ▾

Cohere специально обучила модель так, чтобы документы и вопросы к ним «тяготели» друг к другу в векторном пространстве — но при условии, что они обрабатываются через правильный режим.

Представь два вектора:

  • Документ «Наши склады расположены в Москве и Санкт-Петербурге», обработанный в режиме search_document.
  • Вопрос «где у вас склады?», обработанный в режиме search_query.

Эти два вектора будут близки. Именно это нам и нужно — вопрос должен «притягивать» релевантные документы.

Если оба обрабатывать через search_document — вопрос «попадает» в другое облако векторного пространства, и расстояния до документов будут хуже. Recall падает на 5–10% — незаметно на простых запросах, ощутимо на сложных.

Мнемоника: WF1 загружает документы → search_document. WF2 ищет по запросу пользователя → search_query.

5 Перезалей KB через WF1 в самом конце

Почему нужно: в шаге 1 ты удалил коллекцию — старые векторы пропали. Теперь ноды настроены на Cohere. Нужно снова загрузить все файлы KB через WF1 — они будут записаны уже как 1024-мерные Cohere-векторы.

Как запустить
  1. Открой WF1 Chat Trigger Ingestion.
  2. Нажми Chat (нижняя панель) — откроется встроенный чат n8n.
  3. Прикрепи каждый файл KB и отправь. Если файлов много — по одному или пачкой, как удобно.
  4. После каждого файла смотри в Executions — должен быть Success.

⚠
Типичные ошибки
Три места где обычно застревают
  1. input_type одинаковый в WF1 и WF2. Самая частая ошибка. WF1 = search_document, WF2 = search_query. Разные.
  2. Не пересоздали коллекцию перед сменой нод. Получишь Wrong input: expected dim 1536, got 1024 при первой же загрузке файла.
  3. Native ноды Cohere нет в n8n. Зависит от версии. Если в списке sub-нод нет Embeddings Cohere — нужен HTTP Request, это отдельный более сложный сценарий (~2 часа работы). Уточни у ментора.
